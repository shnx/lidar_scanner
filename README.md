# SLAM Launcher

A production-grade desktop application for Ubuntu 18.04 + ROS Melodic that acts as a unified launcher, orchestrator, and visualisation platform for multiple SLAM packages.

---

## Supported Packages

| Package | Description | ROS Master Port |
|---------|-------------|-----------------|
| **R3LIVE** | LiDAR-Inertial-Visual SLAM в†’ dense coloured point cloud + trajectory | 11311 |
| **livox_camera_calib** | Extrinsic calibration tool в†’ YAML matrix output | 11312 |
| **SR-LIVO** | LiDAR-Visual-Inertial odometry в†’ pose + point cloud | 11313 |
| **ORB-SLAM3** | RGB-D SLAM в†’ trajectory + sparse feature map | 11314 |

Each package runs under its **own `roscore` instance** (distinct port), so topics and TF frames never collide between sessions.

## Installation

```bash
git clone <repo> slam_launcher
cd slam_launcher
chmod +x install.sh
./install.sh
```

`install.sh` will:
1. Install apt packages (PyQt5, OpenGL libraries, вЂ¦)
2. `pip3 install -r requirements.txt`
3. Make all scripts executable
4. Generate the application icon
5. Create `~/.local/share/applications/slam_launcher.desktop`
6. Create `~/slam_sessions/` recording directory

---

## Running

```bash
# Default (English)
python3 main.py

# Arabic UI
python3 main.py --lang ar

# Verbose debug output
python3 main.py --loglevel DEBUG
```

Or double-click the **SLAM Launcher** icon in the application menu.

## Session Data

Each recording is saved to:

```
~/slam_sessions/<pkg_id>_YYYYMMDD_HHMMSS/
    recording.bag       # rosbag of all subscribed topics
    trajectory.txt      # TUM-format poses (timestamp tx ty tz qx qy qz qw)
    pointcloud.pcd      # Final point cloud (binary PCD v0.7)
    config.yaml         # Session metadata + package config snapshot
```

## Customising Package Topics

Edit `config/packages.yaml` вЂ” each package entry has an `outputs` section:

```yaml
outputs:
  pointcloud: "/r3live/cloud_registered"   # sensor_msgs/PointCloud2
  odometry:   "/r3live/odometry"           # nav_msgs/Odometry
  image:      "/r3live/camera/image"       # sensor_msgs/Image
```

Change these to match the actual topic names in your installation.

---

## Workspace Isolation

Each package's `roscore` binds to a unique port (11311вЂ“11314).  
`ROS_MASTER_URI` is overridden per session in `ProcessManager._build_env()`.  
No two sessions can run simultaneously вЂ” the previous session is fully
terminated (SIGTERM в†’ SIGKILL) before a new one starts.

---

## Auto-Recovery

If a SLAM node crashes, `NodeWatchdog` detects it within 2 seconds and
attempts to restart the session (up to 3 times, with a 5-second delay).  
All thresholds are configurable in `config/settings.yaml`.

---

## Adding a New Package

1. Add an entry to `config/packages.yaml` with a unique `rosmaster_port`.
2. Create `scripts/launch_<pkg_id>.sh` following the existing pattern.
3. Build the package in its own catkin workspace at `/opt/ros_workspaces/<pkg>_ws`.
4. Restart the launcher вЂ” the new card appears automatically.
